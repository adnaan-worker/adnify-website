# Adnify 官网图片需求清单

> 本文件列出官网所有需要展示图片的位置，请根据以下清单准备图片资源。

---

## 📁 图片存放路径

所有图片请存放在 `public/images/` 目录下。

---

## 🖼️ 图片清单

### 1. Hero 区域主图

**文件名**: `hero-screenshot.png`

**用途**: Hero 区域的产品主界面展示

**尺寸建议**: 1920×1200px (16:10 比例)

**内容要求**:
- Adnify IDE 主界面完整截图
- 展示代码编辑器区域
- 展示 AI 聊天面板
- 展示文件资源管理器
- 使用默认主题（Adnify Dark）
- 代码内容可以是示例代码（如 TypeScript/React）

**示例截图内容**:
```
- 左侧：文件树（src/components/...）
- 中间：代码编辑器（显示 syntax highlighting）
- 右侧：AI 聊天面板（显示对话历史）
- 底部：终端面板
```

---

### 2. Features 区域 - 主题展示

**文件名**: `feature-themes.png`

**用途**: 展示多主题支持特性

**尺寸建议**: 1920×1080px (16:9 比例)

**内容要求**:
- 展示 4 个主题的对比
- 可以是 4 个窗口并排，或 Tab 切换效果
- 主题包括：Adnify Dark、Midnight、Cyberpunk、Dawn

---

### 3. Features 区域 - AI Agent

**文件名**: `feature-agent.png`

**用途**: 展示 AI Agent 工作模式

**尺寸建议**: 1920×1080px (16:9 比例)

**内容要求**:
- 展示 AI 聊天界面
- 显示工具调用过程（如文件读取、代码编辑）
- 展示 Agent Mode 标签页
- 可以显示一个实际的 AI 对话场景

---

### 4. Features 区域 - 集成终端

**文件名**: `feature-terminal.png`

**用途**: 展示集成终端功能

**尺寸建议**: 1920×1080px (16:9 比例)

**内容要求**:
- 展示终端面板
- 显示命令执行输出
- 展示分屏终端（如果有）

### 5. Features 区域 - Git 支持

**文件名**: `feature-git.png`

**用途**: 展示 Git 版本控制功能

**尺寸建议**: 1920×1080px (16:9 比例)

**内容要求**:
- 展示 Git 面板

- 显示 Diff 对比视图
- 展示提交历史

## 📋 图片规格要求

| 属性 | 要求 |
|------|------|
| **格式** | PNG（推荐）或 WebP |
| **色彩模式** | RGB |
| **分辨率** | 72-144 DPI |
| **文件大小** | 单张不超过 500KB |
| **命名规范** | 小写，使用连字符分隔 |

---

## 🎨 视觉风格建议

1. **截图风格统一**
   - 使用相同的代码示例项目
   - 保持窗口大小一致
   - 字体大小适中（14-16px）

2. **内容美化**
   - 代码要有 syntax highlighting
   - 确保界面没有错误提示
   - 可以隐藏敏感信息（如文件路径中的用户名）

3. **背景处理**
   - 截图可以带窗口边框（macOS 风格的三色按钮）
   - 或使用纯净截图，官网组件会添加窗口装饰

## 💡 临时替代方案

如果暂时没有真实截图，可以使用以下替代方案：

1. **使用占位图**: 官网代码已内置渐变背景作为 fallback
2. **使用 CodeEditor 组件**: Hero 区域保留了原有的代码编辑器组件作为备选
3. **逐步替换**: 可以先上线，后续再补充真实截图
